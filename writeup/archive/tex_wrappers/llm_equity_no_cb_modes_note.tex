\section*{LLM Equity Analysis (Job-Posting Universe): Matched vs Baseline Backfill}

\subsection*{High-level design}
This note runs the equity analysis directly on the firm and user panels after merging LLM equity signals from job postings. The design reports two side-by-side sample modes:
\begin{enumerate}
  \item \textbf{Matched mode (observed-signal sample):} keep firm$\times$half-year rows with parsed LLM coverage (\texttt{llm\_n\_parse\_ok>0}).
  \item \textbf{Baseline backfill mode (full-panel sample):} keep the full panel and code cells without parsed LLM coverage as non-equity.
\end{enumerate}
Backfill uses the coding assumption that, within the submitted candidate-posting universe, absence of parsed equity signal is treated as zero equity exposure.

\subsection*{Data pipeline walkthrough}
\begin{itemize}
  \item Start from postings with non-empty descriptions; apply the deterministic equity-language candidate filter before LLM submission.
  \item Run JSON extraction, aggregate posting outputs to firm$\times$half-year, and merge to outcome panels on normalized \texttt{companyname}$\times$\texttt{yh}.
  \item Define matched as parse-covered rows only; define backfill as full panel with non-parse rows coded as zero equity.
\end{itemize}

\subsection*{Equity variables in tables}
\begin{itemize}
  \item \textbf{Equity firm (main):} binary indicator for whether a firm shows any equity-compensation mention in the analyzed posting universe.
  \item \textbf{Equity share (appendix robustness):} fraction of a firm’s analyzed postings that mention equity compensation.
  \item \textbf{Equity count (appendix robustness):} count-based measure of how frequently a firm’s postings mention equity compensation.
\end{itemize}
